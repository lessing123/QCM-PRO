1.2 Démarrage des serveurs

Première installation

ℹ️ À effectuer une seule fois lors de la première utilisation.
  1. Ouvrez un terminal dans le dossier QCM/ et installez les dépendances backend :
    Terminal → dossier backend
    cd backend && npm install
  2. Initialisez la base de données et injectez les données de test :
    Toujours dans backend/
    npx prisma db push
    npm run seed
  3. Dans un second terminal, installez les dépendances frontend :
    Terminal → dossier frontend
    cd frontend && npm install

Démarrage quotidien

  1. Terminal 1 — Démarrer le serveur backend (API) :
    cd backend && npm run dev
    ✅ Le serveur démarre sur http://localhost:3001
  2. Terminal 2 — Démarrer le serveur frontend :
    cd frontend && npm run dev
    ✅ L'application démarre sur http://localhost:5173

👨‍💼 3. Guide administrateur

3.1 Tableau de bord

Après connexion avec le compte admin, vous accédez au tableau de bord qui affiche :

  • Le nombre total d'examens créés
  • Le nombre d'étudiants enregistrés
  • Le nombre de tentatives effectuées
  • Le score moyen sur l'ensemble de la plateforme
  • Les 5 examens les plus récents
  • Les 10 dernières tentatives terminées

3.2 Créer un examen

  1. Aller dans Examens → Nouvel examen via la navigation.
  2. Remplir le formulaire : titre, description, durée (en minutes), nombre de tentatives autorisées.
  3. Cocher Mélanger les questions et/ou Mélanger les réponses si souhaité.
  4. Assigner un ou plusieurs groupes d'étudiants (optionnel — sans groupe, l'examen est visible par tous).
  5. Ajouter des questions via le bouton + Ajouter une question :
    • SINGLE — une seule réponse correcte
    • MULTIPLE — plusieurs réponses correctes
    • TRUE_FALSE — Vrai / Faux
  6. Cliquer sur Créer l'examen pour enregistrer.

3.3 Gestion des étudiants

Accessible via Étudiants dans la barre de navigation :

Action Description
Créer un étudiant Remplir le formulaire avec email, nom, prénom et mot de passe
Modifier un étudiant Cliquer sur l'icône ✏️ dans le tableau des étudiants
Supprimer un étudiant Cliquer sur l'icône 🗑️ — une confirmation est demandée
Import CSV Préparer un fichier CSV avec colonnes : email,nom,prenom
ℹ️ Format CSV d'import : première ligne = en-tête (email,nom,prenom), une ligne par étudiant. Le mot de passe est généré automatiquement et affiché dans la réponse de l'API.

3.4 Consulter les résultats

  1. Aller dans Examens et cliquer sur le bouton Résultats d'un examen.
  2. Visualiser le tableau avec les scores de chaque étudiant, la date, le temps de passage.
  3. Utiliser le bouton Exporter CSV pour télécharger les résultats au format tableur.
  4. Accéder aux Statistiques pour voir les graphiques de distribution et le taux de réussite par question.
🎓 4. Guide étudiant

4.1 Connexion et tableau de bord

  1. Ouvrir http://localhost:5173 dans le navigateur.
  2. Saisir l'email et le mot de passe du compte étudiant, puis cliquer sur Se connecter.
  3. Le tableau de bord affiche tous les examens disponibles avec leur statut : Disponible, À venir ou Terminé.

4.2 Passer un examen

  1. Cliquer sur le titre d'un examen disponible pour accéder à la page d'instructions.
  2. Lire les instructions : durée, nombre de questions, tentatives autorisées.
  3. Cliquer sur Commencer l'examen.
  4. Répondre aux questions. Le timer en haut à droite affiche le temps restant en rouge lorsqu'il reste moins de 5 minutes.
  5. Naviguer entre les questions via la sidebar gauche : numéros verts = répondus, gris = non répondus, bordure jaune = marqués pour révision.
  6. Cliquer sur ☆ Marquer pour signaler une question à revoir.
  7. Cliquer sur Soumettre → confirmer dans le modal récapitulatif.
⚠️ Anti-triche : Le système détecte les changements d'onglet. Un avertissement s'affiche si vous quittez l'onglet d'examen. Le clic droit et le copier-coller sont désactivés pendant l'examen.

🧪 5. Scénarios de test
ℹ️ Cochez chaque case après validation pour suivre votre progression de test.

5.1 Tests côté administrateur

ADMIN Authentification

  • Se connecter avec admin@test.com / admin123 → redirection vers /admin/dashboard
  • Tenter une connexion avec un mot de passe incorrect → message d'erreur affiché
  • Se déconnecter → redirection vers la page de login

ADMIN Tableau de bord

  • Vérifier l'affichage de : 2 examens, 5 étudiants, tentatives, score moyen
  • Vérifier la liste des examens récents et des tentatives récentes
  • Cliquer sur un examen récent → accès aux résultats

ADMIN Gestion des examens

  • Créer un nouvel examen avec au moins 3 questions et 4 réponses chacune
  • Vérifier qu'on ne peut pas enregistrer sans réponse correcte
  • Modifier le titre et la durée d'un examen existant
  • Dupliquer un examen → vérifier que "(copie)" est ajouté
  • Supprimer l'examen dupliqué (confirmation demandée)
  • Assigner un examen à un groupe d'étudiants

ADMIN Gestion des étudiants

  • Créer un nouvel étudiant avec un email unique
  • Tenter de créer un étudiant avec un email déjà utilisé → erreur 409
  • Modifier le nom d'un étudiant existant
  • Supprimer l'étudiant créé

ADMIN Gestion des groupes

  • Créer un groupe avec un nom unique
  • Ajouter plusieurs étudiants au groupe
  • Vérifier que les étudiants du groupe voient l'examen assigné
  • Supprimer le groupe créé

ADMIN Résultats et export

  • Voir les résultats de l'examen "Introduction à la Programmation"
  • Cliquer sur Exporter CSV → fichier téléchargé avec les colonnes correctes
  • Consulter les statistiques → graphiques et taux de réussite par question

5.2 Tests côté étudiant

ÉTUDIANT Passer un examen complet

  • Se connecter avec etudiant2@test.com / student123
  • Voir les 2 examens disponibles dans le tableau de bord
  • Cliquer sur "Introduction à la Programmation" → page d'instructions affichée
  • Démarrer l'examen → timer affiché, questions chargées
  • Répondre à toutes les questions en naviguant via la sidebar
  • Marquer 2 questions pour révision → bordure jaune visible
  • Vérifier que les réponses sont sauvegardées automatiquement
  • Soumettre → modal de confirmation avec récapitulatif
  • Confirmer → affichage de la page résultats avec le score

ÉTUDIANT Timer et soumission automatique

  • Créer un examen de 1 minute (admin) → passer cet examen et attendre la fin
  • Vérifier que l'examen est soumis automatiquement à l'expiration du timer
  • Vérifier que le timer passe en rouge quand il reste moins de 5 minutes

ÉTUDIANT Historique

  • Aller dans "Historique" → toutes les tentatives passées sont listées
  • Cliquer sur une tentative → accès au détail des résultats

5.3 Tests de sécurité

  • Pendant un examen, changer d'onglet → alerte anti-triche affichée
  • Pendant un examen, faire clic droit → menu contextuel bloqué
  • Tenter d'accéder à /admin/dashboard avec un compte étudiant → redirection
  • Tenter d'accéder à /student/dashboard avec un compte admin → redirection
  • Accéder à une URL protégée sans être connecté → redirection vers /login
  • Vérifier que l'examen d'un étudiant ayant atteint le nombre max de tentatives indique "Terminé"
  • Vérifier que le score est calculé côté serveur (impossible à manipuler côté client)

❓ 7. FAQ & Dépannage

Problèmes courants

SymptômeCause probableSolution
Le backend ne démarre pas Port 3001 déjà occupé Changer PORT dans backend/.env
Erreur "Cannot find module" npm install pas effectué Relancer npm install dans backend/ et frontend/
La base de données est vide Seed pas exécuté Dans backend/ : npm run seed
Erreur 401 à la connexion Token expiré ou invalide Se déconnecter et se reconnecter
Le frontend affiche une page blanche Erreur JavaScript Ouvrir la console navigateur (F12) pour voir l'erreur
L'API retourne 500 Erreur serveur Consulter les logs dans le terminal du backend

Variables d'environnement

backend/.env
DATABASE_URL="file:./dev.db"
JWT_SECRET="votre-secret-jwt"
JWT_REFRESH_SECRET="votre-refresh-secret"
JWT_EXPIRES_IN="1h"
JWT_REFRESH_EXPIRES_IN="7d"
PORT=3001
✅ Réinitialiser complètement la base de données :
Supprimer le fichier backend/prisma/dev.db, puis relancer :
npx prisma db push && npm run seed
